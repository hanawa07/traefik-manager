# 릴리즈 노트 가이드

이 저장소의 GitHub Release 본문은 아래 규칙으로 고정합니다.

## 목적

- 릴리즈마다 제목과 섹션 이름이 바뀌지 않게 유지
- 사용자 관점의 변경만 짧게 전달
- 커밋 로그, 검증 로그, 영어 템플릿이 섞이는 문제 방지

## 표준 템플릿

```markdown
## 주요 변경
- ...
- ...
```

## 작성 규칙

- 본문 언어는 한국어로 고정합니다.
- 섹션 제목은 항상 `## 주요 변경`만 사용합니다.
- bullet은 1개 이상 4개 이하로 유지합니다.
- 각 bullet은 한 문장으로 끝냅니다.
- 커밋 제목을 그대로 복사하지 말고, 사용자 관점 문장으로 다시 씁니다.
- 문서/테스트/리팩터링/CI 변경은 사용자 체감 변화가 없으면 제외합니다.
- 여러 커밋이 같은 기능을 설명하면 한 bullet로 합칩니다.

## 금지 규칙

- `## 변경 사항`
- `## Summary`
- `**Full Changelog**`만 넣는 형식
- `## 포함 커밋`
- `## 검증`
- 커밋 해시 나열
- 영어와 한국어 혼용

## 좋은 예시

```markdown
## 주요 변경
- Cloudflare DNS 자동 연동을 멀티존 구조까지 처리하도록 확장했습니다.
- Cloudflare를 사용하지 않는 도메인은 드리프트 진단과 재동기화 대상에서 자동 제외합니다.
- 설정 화면에서 zone별 상태와 제외 사유를 함께 확인할 수 있습니다.
```

## 나쁜 예시

```markdown
## Summary
- Add multi-zone support

## 포함 커밋
- 21401f0 설정: Cloudflare 멀티존과 비대상 도메인 제외 지원
```

## 생성 순서

1. 직전 릴리즈 이후 변경만 모읍니다.
2. 사용자에게 보이는 변화만 남깁니다.
3. 겹치는 항목을 합쳐 1~4개 bullet로 줄입니다.
4. 위 템플릿 그대로 `gh release create ... --notes-file`에 넣습니다.
